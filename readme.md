# Шаблон Репозитория Проекта

-----

## Описание

Этот репозиторий служит **шаблоном** для быстрого старта новых проектов. Он содержит набор **типовых файлов и конфигураций**, которые являются неотъемлемой частью большинства современных репозиториев, обеспечивая согласованность, безопасность и эффективность с самого начала.

Включенные типовые файлы:

  * `.gitignore`: Стандартный файл для игнорирования ненужных файлов и папок в Git.
  * `.gitlab-ci.yml`: Базовый шаблон для настройки непрерывной интеграции и доставки (CI/CD) в GitLab.
  * `.gitleaks.toml`: Конфигурация для Gitleaks, инструмента для обнаружения секретов в коде.
  * `CONTRIBUTING.md`: Руководство для участников, описывающее процесс внесения изменений в проект.
  * `LICENSE`: Файл лицензии, определяющий условия использования и распространения кода.

Использование этого шаблона поможет вам:

  * **Ускорить инициализацию проекта**: Не тратьте время на создание базовых файлов с нуля.
  * **Обеспечить стандартизацию**: Поддерживайте единые правила и практики во всех ваших проектах.
  * **Улучшить безопасность**: Включите проверку на утечки секретов с самого начала.
  * **Упростить вклад**: Предоставьте четкие инструкции для будущих участников.

-----

## Как Использовать Этот Шаблон

Есть несколько способов использовать этот репозиторий в качестве шаблона для вашего нового проекта:

### 1\. Использование функции "Use template" в GitLab

Это **рекомендуемый** способ, так как он создает новый репозиторий на основе этого шаблона, сохраняя его чистым и готовым к работе.

1.  Перейдите на страницу этого репозитория в GitLab.
2.  Нажмите кнопку **"Use template"** (Использовать шаблон).
3.  Следуйте инструкциям GitLab для создания нового проекта на основе этого шаблона.

### 2\. Клонирование и очистка истории Git

Если вы предпочитаете ручной подход или ваша платформа не поддерживает функцию шаблонов:

1.  Клонируйте этот репозиторий:
    ```bash
    git clone https://gitlab.okbtsp.corp/oit/template.git my-new-project
    ```
2.  Перейдите в новую папку проекта:
    ```bash
    cd my-new-project
    ```
3.  Удалите существующую историю Git:
    ```bash
    rm -rf .git
    ```
4.  Инициализируйте новый репозиторий Git:
    ```bash
    git init
    ```
5.  Добавьте и закоммитьте файлы:
    ```bash
    git add .
    git commit -m "Initial commit from template"
    ```
6.  Свяжите новый локальный репозиторий с вашим удаленным репозиторием (создайте его в GitLab, если еще не сделали):
    ```bash
    git remote add origin <URL_вашего_нового_репозитория>
    git push -u origin master
    ```

## Включенные Файлы и Их Назначение

### `.gitignore`

Этот файл указывает Git, какие файлы и папки следует игнорировать и не включать в систему контроля версий. Он содержит типовые записи для операционных систем, сред разработки, временных файлов и зависимостей.

  * **Настройте его**: Обязательно добавьте специфичные для вашего проекта записи (например, папки с логами, файлы конфигурации с чувствительными данными).

### `.gitlab-ci.yml`

Основной файл конфигурации для GitLab CI/CD. Он определяет пайплайны, этапы, задания и зависимости для автоматизации процессов сборки, тестирования и развертывания.

  * **Настройте его**: Этот шаблон предоставляет базовую структуру. Вам потребуется адаптировать его под нужды вашего проекта, добавив специфические шаги для компиляции, выполнения тестов, линтинга и деплоя.

### `.gitleaks.toml`

Конфигурационный файл для [Gitleaks](https://github.com/gitleaks/gitleaks), инструмента, который помогает предотвратить утечку конфиденциальных данных (паролей, ключей API и т.д.) в ваш репозиторий. Он содержит правила для обнаружения распространенных типов секретов.

  * **Настройте его**: Вы можете добавить пользовательские правила или исключения, если у вас есть специфические требования к обнаружению секретов. Рекомендуется интегрировать Gitleaks в ваш CI/CD пайплайн.

### `CONTRIBUTING.md`

Этот файл предоставляет руководство для всех, кто хочет внести свой вклад в ваш проект. Он описывает процесс создания Issues, Merge Requests, стандарты кода, правила именования веток и коммитов, а также ожидания по тестированию.

  * **Настройте его**: Отредактируйте этот файл, чтобы он соответствовал вашим конкретным процессам и стандартам разработки.

### `LICENSE`

Файл лицензии определяет, как другие могут использовать, распространять и изменять ваш код. В этом шаблоне может быть включена популярная лицензия, например, MIT.

  * **Выберите свою лицензию**: Убедитесь, что выбранная лицензия соответствует вашим намерениям. Если вам нужна другая лицензия, замените содержимое этого файла на текст выбранной вами лицензии (например, Apache 2.0, GPLv3 и т.д.).
